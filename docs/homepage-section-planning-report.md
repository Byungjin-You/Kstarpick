# KstarPick 홈페이지 섹션 구성 기획서

**작성일:** 2026.02.23
**작성자:** 서비스기획팀
**문서 구분:** 홈페이지 개편 기획안 (v1.0)

---

## 1. 개요

### 1.1 목적
KstarPick 홈페이지의 각 섹션별 콘텐츠 표출 기준을 수립하여,
**유저가 이탈하지 않고 뉴스를 연속적으로 소비하도록 유도**하는 것을 목표로 한다.

### 1.2 배경
현재 홈페이지는 최신 기사 20개를 순번(0~4, 5~7, 8~10, 11~)으로 슬라이싱하여
이름만 다른 섹션에 배분하고 있어, 섹션 간 차별성이 없고 동일 기사가 중복 노출되는 문제가 있다.

---

## 2. 현황 분석

### 2.1 콘텐츠 현황 (최근 300건 기준)

| 항목 | 수치 | 비고 |
|------|------|------|
| 전체 기사 | 300건 | 2/6 ~ 2/23, 약 17일간 |
| 일 평균 발행 | ~18건/일 | |
| 카테고리 | Drama 57.7%, K-pop 20.0%, Celeb 16.0%, Movie 6.3% | Variety 0% |
| Featured 기사 | 213건 (71%) | 대부분이 featured → 선별 의미 희석 |
| Watch 뉴스 | 25건 (8.3%) | 영상 콘텐츠 |
| 평균 조회수 | 48.3회 | Max 229, Median 48 |

### 2.2 현재 홈페이지 섹션 구성 및 문제점

| 순서 | 섹션명 | 데이터 소스 | 개수 | 문제점 |
|------|--------|------------|------|--------|
| 1 | Comment Ticker | recentComments | 롤링 | - |
| 2 | Hero + Hot News | articles[0:5] | 5 | 단순 최신순, 중요도 반영 없음 |
| 3 | Trending NOW | todayRankingNews (viewCount) | 5 | Hero와 중복 가능 |
| 4 | Latest News | articles[5:8] | 3 | Hero와 같은 풀에서 순번 슬라이싱 |
| 5 | Recommended | featured[0:3] 또는 articles[8:11] | 3 | Hero와 80% 중복 (featured 비율 71%) |
| 6 | Watch News | watchNews (title 키워드 필터) | 6 | - |
| 7 | K-POP Ranking | topSongs (별도 컬렉션) | 3~4 | - |
| 8 | More News | articles[11:] | 무한스크롤 | 위 섹션 잔여분으로 시작 |

#### 핵심 문제

**① 동일 데이터풀 돌려막기**
- articles 배열 하나를 [0:5], [5:8], [8:11], [11:]로 잘라서 4개 섹션에 배분
- 유저 관점에서 Latest News와 Recommended의 차이를 체감할 수 없음

**② 중복 노출**
- Hero(상위 5개)와 Editor's PICK(featured 상위 5개) 간 중복: **4~5건** (80~100%)
- featured 비율이 71%로 과도하게 높아 "에디터 선정"의 의미가 희석

**③ 콘텐츠 소비 동기 부재**
- 모든 섹션이 동일한 카드 UI → "왜 이 기사를 여기서 봐야 하는지" 맥락 없음
- 조회수, 반응, 시간 등 어떤 신호도 표출 기준에 반영되지 않음

## 3. 개선안

### 3.1 섹션 구성 및 표출 기준

| 순서 | 섹션명 | 개수 | 표출 기준 | 유저 인식 목표 |
|------|--------|------|----------|---------------|
| 1 | **Comment Ticker** | 롤링 | 최근 24h 댓글 중 최신순 | "사람들이 지금 반응하고 있다" |
| 2 | **Hero + Hot News** | **1+4=5** | viewCount 기반 상위 5개 | "지금 가장 많이 읽히는 기사" |
| 3 | **Trending NOW** | 5 | 최근 3일 조회수 상위 (별도 API) | "급상승 중인 기사" |
| 4 | **Latest News** | 3 | createdAt DESC, 위 중복 제외 | "방금 올라온 따끈한 기사" |
| 5 | **Editor's PICK** | **5** | featured=true, 위 중복 제외 | "에디터가 직접 고른 기사" |
| 6 | **Watch News** | 6 | title에 "Watch:" 포함, 최근 3일 | "영상으로 보는 뉴스" (포맷 전환) |
| 7 | **K-POP Ranking** | 4(MO)/3(PC) | 음악 차트 데이터 (별도 컬렉션) | "음악 콘텐츠" (타입 전환) |
| 8 | **More News** | 무한스크롤 | 위 전체 ID 제외, 최신순 | "끝없이 이어지는 뉴스 피드" |

### 3.2 중복 제거 체인 (Deduplication Chain)

각 섹션은 이전 섹션에서 사용된 기사를 제외하고 선택한다.

```
usedIds = []

① Hero + Hot News  → viewCount 상위 5개 선택 → usedIds에 추가
② Trending NOW     → 별도 API (todayRankingNews), 프론트에서 usedIds 표출 제외
③ Latest News      → createdAt DESC, usedIds 제외 후 3개 → usedIds에 추가
④ Editor's PICK    → featured=true, usedIds 제외 후 5개 → usedIds에 추가
⑤ Watch News       → title 필터, usedIds 제외 후 6개 → usedIds에 추가
⑥ More News        → usedIds 전체 제외 후 무한스크롤 시작
```

**시뮬레이션 결과 (실제 300건 데이터 기준):**

| 섹션 | 사용 기사 수 | 누적 usedIds | 중복 |
|------|------------|-------------|------|
| Hero + Hot News | 5 | 5 | 0 |
| Latest News | 3 | 8 | 0 |
| Editor's PICK | 5 | 13 | 0 |
| Watch News | 6 | 19 | 0 |
| **More News 시작** | - | **281건 잔여** | **0** |

### 3.3 Hero 선정 기준 상세

**현재:** 단순 createdAt DESC (가장 최근 기사가 1위)
**개선:** viewCount 기반 정렬

```
정렬 기준: viewCount DESC
```

| 순위 | 기존 (최신순) | 개선 (조회수순) |
|------|-------------|---------------|
| 1위 (Hero) | 직전 발행 기사 (8 views) | Bae In Hyuk 가족 케미 기사 (229 views) |
| 2위 | 2번째 최근 기사 (7 views) | KiiiKiii 2nd Win (106 views) |
| 3위 | 3번째 최근 기사 (16 views) | Philippines K-Pop Tour (97 views) |

> **효과:** "많이 읽히는 기사 = 관심 있는 기사"를 상단에 배치하여 클릭률 향상 기대

### 3.4 Editor's PICK 운영 개선 제안

**현재 문제:** featured=true 기사가 전체의 71% (213/300건)
→ 거의 모든 기사가 featured이므로 "에디터가 골랐다"는 차별성이 없음

**개선 방향:**
- featured 비율을 **전체의 10~15% 수준**으로 조정 (하루 2~3건)
- 명확한 선정 기준 가이드 수립:
  - 독점/단독 기사
  - 깊이 있는 분석 기사
  - 커뮤니티 반응이 특별히 높은 기사
- Admin 페이지에서 featured 토글 시 선정 사유 입력 필드 추가 (향후)

## 4. 섹션 간 흐름 설계 (유저 저니)

```
┌─────────────────────────────────────────────────────────┐
│  Comment Ticker (롤링)                                    │
│  "지금 이 기사에 이런 댓글이 달렸어" → 궁금증 유발         │
├─────────────────────────────────────────────────────────┤
│  Hero + Hot News (5)                                     │
│  "가장 많이 읽히는 기사" → 높은 클릭률, 첫 콘텐츠 소비     │
├─────────────────────────────────────────────────────────┤
│  Trending NOW (5)                                        │
│  "갑자기 뜨고 있는 기사" → 시의성, FOMO 자극               │
├─────────────────────────────────────────────────────────┤
│  Latest News (3)                                         │
│  "방금 올라온 기사" → 신선함, 빠른 정보 획득 욕구           │
├─────────────────────────────────────────────────────────┤
│  Editor's PICK (5)                                       │
│  "에디터 추천" → 신뢰감, 놓칠 뻔한 양질의 콘텐츠 발견      │
├─────────────────────────────────────────────────────────┤
│  Watch News (6)                                          │
│  "영상 뉴스" → 포맷 전환으로 스크롤 피로도 해소             │
├─────────────────────────────────────────────────────────┤
│  K-POP Ranking (4)                                       │
│  "음악 차트" → 콘텐츠 타입 전환, 새로운 관심 유도           │
├─────────────────────────────────────────────────────────┤
│  More News (∞)                                           │
│  "끝없는 피드" → 중복 0건, 잔여 281건부터 무한스크롤        │
└─────────────────────────────────────────────────────────┘
```

**설계 의도:**
- **상단(1~3):** 높은 관여도 콘텐츠로 첫 클릭 유도
- **중단(4~5):** 다른 기준의 콘텐츠로 "더 볼 게 있다" 인식
- **하단(6~7):** 텍스트 → 영상 → 음악으로 포맷 전환, 스크롤 피로 완화
- **최하단(8):** 잔여 콘텐츠를 중복 없이 무한 공급

## 5. 구현 범위

### 5.1 프론트엔드 변경 (index.js)

| 변경 사항 | 상세 |
|----------|------|
| Hero 데이터 소스 변경 | articles(최신순) → viewCount 상위 5개 |
| usedIds 중복 제거 로직 추가 | useMemo로 각 섹션별 사용 ID 추적 |
| Recommended → Editor's PICK 변경 | 섹션명 변경, 5개로 확대 (기존 3개) |
| Editor's PICK 레이아웃 | Hero와 동일한 1+4 구조 (HeroSection + NewsCardGrid 재사용) |
| More News exclude 파라미터 | usedIds를 API에 전달하여 서버 측에서도 제외 |

### 5.2 백엔드 변경

| 변경 사항 | 상세 |
|----------|------|
| /api/news exclude 파라미터 | More News 무한스크롤 시 이미 표출된 ID 제외 |

### 5.3 운영 가이드 (에디터)

| 항목 | 가이드 |
|------|--------|
| featured 선정 | 하루 2~3건으로 제한, 독점/분석/고반응 기사 위주 |
| Watch 뉴스 | 제목에 "Watch:" 접두어 필수 |

## 6. 기대 효과

| 지표 | 현재 | 기대 |
|------|------|------|
| 섹션 간 기사 중복률 | 80~100% (Hero↔Editor's PICK) | **0%** |
| Hero 클릭률 | 최신순 = 관심 무관 | 조회수 기반 = 이미 검증된 관심 기사 |
| 평균 스크롤 깊이 | 동일 기사 반복 → 이탈 | 중복 제거 → 새 콘텐츠 계속 노출 |
| More News 시작 잔여량 | ~9건 (20 - 11) | **~281건** (300 - 19) |
| 카테고리 다양성 | 상단 8건 중 Drama 편중 | 자연 분산 + 향후 강제 분산 가능 |

| Hot Score 복합 점수 | viewCount + reactionCount + commentCount + 시간감쇠 | Phase 2 |
| 카테고리 강제 분산 | Editor's PICK에서 미노출 카테고리 우선 선정 | Phase 2 |
