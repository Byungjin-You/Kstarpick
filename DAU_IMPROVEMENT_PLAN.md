# KstarPick DAU 증대 개선 기획안

## 📊 현황 분석

### 현재 사이트 구조
- **콘텐츠**: K-Pop 뉴스, 드라마/영화(TV Film), 셀럽, 음악 차트, 리뷰
- **주요 기능**: 뉴스 읽기, 검색, 음악 차트, 드라마/영화 정보, 좋아요/북마크
- **기술 스택**: Next.js, MongoDB, NextAuth
- **데이터**: 뉴스 2,221건, 드라마 1,204건, 영화 230건, 셀럽 259명

### 현재 문제점 분석
1. **재방문 유도 요소 부족**: 한 번 보고 나가는 구조
2. **개인화 부재**: 모든 사용자에게 동일한 콘텐츠 제공
3. **사용자 참여도 낮음**: 읽기만 가능, 커뮤니티 요소 부족
4. **푸시/알림 시스템 없음**: 신규 콘텐츠 알림 불가
5. **소셜 기능 미비**: 공유, 댓글, 토론 기능 제한적

---

## 🎯 DAU 증대 전략 (3단계 로드맵)

### Phase 1: Quick Wins (1-2주, 즉시 효과)
빠르게 구현 가능하고 즉각적인 효과를 볼 수 있는 기능

### Phase 2: Core Features (1-2개월, 중기 효과)
핵심 재방문율 증대 기능

### Phase 3: Long-term Growth (3-6개월, 장기 성장)
커뮤니티 활성화 및 네트워크 효과

## 🚀 Phase 1: Quick Wins (우선순위 높음)

### 1.1 푸시 알림 시스템 구축 ⭐⭐⭐⭐⭐
**목표**: 신규 콘텐츠 발행 시 사용자 재방문 유도

**구현 내용**:
- 브라우저 푸시 알림 (PWA)
- 관심 아티스트/드라마 알림 설정
- 인기 뉴스 알림 (조회수 급상승)
- 음악 차트 업데이트 알림

**구현 방법**:
```javascript
// 1. Service Worker 등록
// public/sw.js 파일 생성

// 2. Push API 구독
// components/PushNotification.js
- OneSignal 또는 Firebase Cloud Messaging 활용
- 사용자 알림 설정 페이지 추가

// 3. 서버 측 트리거
// pages/api/notifications/send.js
- 새 뉴스 발행 시 자동 발송
- 스케줄러로 주요 시간대 발송
```

**기대 효과**:
- DAU +25-40% (푸시 알림 평균 클릭률 5-15%)
- 재방문율 +30%

**구현 난이도**: ⭐⭐ (중)
**개발 기간**: 1주일

### 1.2 개인화 홈피드 시스템 ⭐⭐⭐⭐⭐
**목표**: 사용자별 맞춤 콘텐츠로 체류 시간 증가

**구현 내용**:
- 사용자 관심사 기반 뉴스 추천
- 최근 본 아티스트/드라마 관련 콘텐츠 우선 노출
- "회원님을 위한 추천" 섹션 추가
- 읽지 않은 콘텐츠 우선 표시

**구현 방법**:
```javascript
// 1. 사용자 행동 추적
// lib/analytics/userBehavior.js
- 클릭한 뉴스 카테고리
- 관심 아티스트 (좋아요, 북마크 기반)
- 체류 시간이 긴 콘텐츠

// 2. 추천 알고리즘
// lib/recommendation/engine.js
- Collaborative Filtering (협업 필터링)
- Content-based Filtering (콘텐츠 기반)
- Hybrid 방식

// 3. 개인화 피드 API
// pages/api/feed/personalized.js
- 사용자별 추천 뉴스 반환
```

**기대 효과**:
- 평균 세션 시간 +40%
- 페이지뷰 +50%
- DAU +15-20%

**구현 난이도**: ⭐⭐⭐ (중상)
**개발 기간**: 2주일
